Why is Exploration in design Important?

In my opinion Exploration is a very critical part of the design process. It allows us designers to get our ideas out when we're thinking through problems [no matter how crazy these ideas can be sometimes]. We can then talk about and review our explorations together and see if there are any directions that we can go that will push us further to a solution. A mantra of this phase of the process should be; With every direction you go, you take one step closer to the solution.'


September | 2019

Ghost Inputs & Labels.


The Problem: (Part A) The inputs we currently have in Radius are not very flexible when it comes to a handheld device like an iPad. The current labels were small and hard to read when we tested them with our clients. (Part B) We also discussed moving away from the label completely and just relying on the placeholder text to help create context for the given input. BUT...on focus states of the input, user could lose context of what the input related to. The goal of this exploration was to try and play around with the type size of the label and the place holder text. I added an animation to the input as well. The default state of the inputs would in fact NOT present a label, on focus the label appears at the top of the input field. See examples below.

I’m technically not part of the Radius team, but I decided to pick up this quick project and do some exploration on how this component would function in our products, what the sates would be and how it could adapt if there were multi selections needed.


Its extremely important to view these components in a high-fidelity prototype to see how flexible they are/can be for the devices we're designing for. It also allows stakeholders see these inputs in context to where they will live in the products we design. I put these inputs to work in a real world setting for a password reset workflow.
